Watmonitor: interfaz web para medidor de nivel

Watmonitor es una aplicación web centralizada para la visualización de niveles de líquidos y materiales a granel. Permite conectar sensores mediante API y almacenar datos en una base de datos. Gracias a su interfaz intuitiva, puede monitorizar el nivel y el volumen actuales en tiempo real, incluyendo tendencias (subidas y bajadas) y el estado de la conexión.

Características principales

  • Valores de nivel y volumen del tanque en tiempo real con tendencias de medición
  • Visualizaciones avanzadas: gráficos con historial de mediciones
  • Exportación de datos de gráficos a formatos .svg, .png y .csv
  • Valores de nivel máximo y mínimo por período
  • Compatibilidad con mediciones de nivel diferencial y total
  • Diseño adaptable: compatible con PC y smartphone
  • Posibilidad de integración con otros sistemas mediante terminales JSON

La solución está diseñada para monitorizar agua, aceite, combustible, biomasa, pellets, semillas, piensos, gránulos y otros materiales.

Escenarios de Uso

Watmonitor es ideal para el monitoreo de nivel de agua en tiempo real en pozos, tanques de agua, sumideros, lagos, estanques, ríos, embalses y sistemas de alcantarillado. También es adecuado para gestión de aguas residuales, gestión industrial del agua y medición de niveles de materiales a granel tanto en aplicaciones industriales como residenciales.

Tipos de Sensores Compatibles

Watmonitor ofrece una interfaz web universal compatible con cualquier sensor. Integra automáticamente datos de sensores ultrasónicos, sensores de distancia láser o cualquier sensor IoT personalizado con firmware configurado para el sistema. Para soluciones industriales, se pueden conectar sensores directamente a través de la API de Watmonitor para monitoreo y recopilación de datos en tiempo real.

Prototipado Rápido

Para un inicio rápido con Watmonitor, están disponibles códigos fuente para hardware de código abierto tipo DIY. Esto incluye ESP32, ESP8266, Raspberry Pi Pico W (también en Micropython), Arduino con soporte WiFi o Ethernet y sensores de distancia ultrasónicos o láser ToF. Se pueden encontrar implementaciones básicas o más avanzadas, incluyendo funcionalidades como bajo consumo de energía (Ultra Low Power) y actualizaciones Over-The-Air (OTA). Para ESP32, también hay implementaciones disponibles en ESP-IDF, con FreeRTOS y comunicación entre tareas (inter-task communication).

Funciones QR / AR

Watmonitor permite el acceso instantáneo a los últimos datos de los sensores mediante la lectura de un código QR con tu smartphone. Los usuarios pueden ver una visualización estática o una escena interactiva en AR (Realidad Aumentada) con distancia de cámara fija, proporcionando una forma más atractiva de monitorear niveles de agua y otros datos de sensores en tiempo real.

PCB de interconexión

Prototipado rápido con hardware disponible comercialmente

Las placas de circuito impreso (PCB) de interconexión permiten el uso de kits de desarrollo WiFi preconfigurados como Lolin32 o XIAO. Posibilidad de alimentación mediante batería de polímero de litio y carga mediante panel solar de 5 V o USB. Compatible con el módulo LoRa RA-02 (433 MHz). La interfaz de 4 pines permite la conexión de sensores ultrasónicos HC-SR04 o JSN-SR04T, sensores láser ToF VL53L1X o VL53L0X, o su propio sensor. El hardware se puede soldar directamente a la PCB o montar en conectores de pines para facilitar su sustitución sin necesidad de soldadura.

Preguntas frecuentes

Las preguntas más frecuentes de la comunidad se responden a continuación

¿Necesito alojamiento web para ejecutar Watmonitor en mi propio servidor?

Sí, para ejecutar Watmonitor de forma independiente, necesitará un alojamiento web local o en Internet. Para ejecutar los scripts PHP para Watmonitor, su servidor debe tener instalado Apache o NGINX. Watmonitor ha sido probado y funciona en las versiones PHP 7 a 8.4.

¿Es difícil configurar la aplicación web para Watmonitor?

La configuración de la aplicación web para Watmonitor es sencilla gracias a una guía paso a paso detallada en formato PDF. Esta guía simplifica el proceso de configuración para todos los usuarios, incluidas las instrucciones para importar el archivo .sql en MySQL, configurar connect.php con sus credenciales de base de datos, configurar los datos de HTTP Auth e ingresar el token del dispositivo.

¿Watmonitor generará automáticamente el código fuente para mi hardware?

Sí, Watmonitor generará automáticamente el código fuente para el ESP32 con Wi-Fi y ULP para los sensores láser ultrasónicos o ToF compatibles. El código fuente generado contendrá el token MCU necesario, la ruta (profundidad) al archivo PHP de destino y la dirección de dominio (nota: localhost no funcionará; asegúrese de usar una dirección IP o un nombre de dominio accesibles). El certificado de CA raíz (para conexiones HTTPS) no se editará, por lo que deberá agregarlo manualmente al código fuente del ESP32. Por defecto, se utiliza el certificado ISRG Root X1 de Let's Encrypt Root CA.

¿Hay otros códigos fuente disponibles además de los que se encuentran en la aplicación Watmonitor?

Sí, hay varios códigos fuente adicionales disponibles en GitHub que son compatibles con la interfaz Watmonitor. Estos incluyen ejemplos para Arduino con Ethernet (serie ENC o Wiznet), ESP8266, ESP32 y otras tecnologías de transmisión como LoRaWAN y Sigfox IoT. Puede acceder a estos códigos fuente haciendo clic en el botón "Ejemplos de nodos de sensores" en la parte superior de esta página.

¿Qué opciones de idioma están disponibles en la aplicación web de Watmonitor?

La aplicación web de Watmonitor ofrece traducciones completas para los siguientes idiomas: inglés, alemán, ruso, español, francés y eslovaco, lo que la hace accesible a una audiencia global.

¿Cómo integrar Watmonitor con otros sistemas?

Watmonitor cuenta con dos puntos finales de API JSON, accesibles mediante HTTP GET:
• json_output.php: proporciona los datos más recientes conocidos (utilizados de forma nativa por escáneres QR y visualización de RA).
• json_output2.php: proporciona todos los datos registrados con la opción de introducir parámetros GET para limitar los resultados (desde-hasta), o todos desde o todos hasta.
Los puntos finales permiten la integración con otros sistemas habilitados para automatización:: Node-RED, Ignition SCADA, NetSuite, AWS IoT Core, ThingsBoard, Grafana, ELK Stack, Power BI, Tableau, Home Assistant ... Ampliable con webhooks para automatización (Zapier, IFTTT, n8n, Microsoft Power Automate).

¿Cómo es la comunicación entre el servidor y el nodo sensor Arduino/ESP32?

El nodo sensor realiza una rutina periódica de medición del nivel del agua cada 5 minutos. Luego realiza una solicitud al servidor web a través del protocolo HTTP o HTTPS con una solicitud POST. La información sobre el nivel del agua se incluye en el informe. Después de recibir el mensaje, el servidor WEB almacena estos datos en la base de datos MySQL y puede visualizarlos inmediatamente en la aplicación web del medidor de nivel de agua (Watmonitor).

¿Cómo se realiza la medición mediante el nodo sensor IoT?

Un nodo sensor basado en Arduino o ESP32 mide la distancia utilizando un sensor ultrasónico con el método de tiempo de vuelo. Envía un nivel de voltaje ALTO al pin TRIGGER del sensor ultrasónico durante 10 μs. Luego, cuenta el tiempo hasta que se detecta un nivel de voltaje ALTO en el pin ECHO del sensor ultrasónico. En función de la velocidad del sonido, se calcula la distancia final en centímetros.

¿Cómo se ve una devolución de llamada en Watmonitor?

Para enviar datos a Watmonitor, realice una solicitud HTTP POST al archivo data.php del servidor (por ejemplo, https://hladinomer.eu/data.php) utilizando el puerto 80 o 443. Incluya dos parámetros: hodnota (diferencia del nivel del agua en cm, INT) y token (autorización). La solicitud debe incluir la cabecera Content-Type: application/x-www-form-urlencoded. Si la escritura se realiza correctamente, se devuelve una respuesta HTTP 200 con OK.
